User Guide

Use pyspice_rs when you want Python code to create SPICE circuits, generate netlists, run a simulator, and inspect numeric results.

The usual workflow is: install, build a circuit, print the netlist, run an analysis, then read nodes or waveforms from the result object.

1. Install

For local development, build the Python extension with maturin. Install at least one simulator if you want to run analyses; ngspice is the simplest starting point.

python3 -m venv .venv
source .venv/bin/activate
pip install maturin numpy pytest
maturin develop

If you use the Nix shell, enter it first:

nix develop

Check that Python can import the package:

import pyspice_rs as ps
print(ps.Circuit("ready"))

2. Build a First Circuit

A circuit is a SPICE deck built from Python calls. Nodes are strings. Ground is always circuit.gnd, which is node "0".

import pyspice_rs as ps
from pyspice_rs.unit import u_V, u_kOhm

circuit = ps.Circuit("voltage_divider")
circuit.V(name="in", positive="vin", negative=circuit.gnd, value=10 @ u_V)
circuit.R(name="top", positive="vin", negative="vout", value=2 @ u_kOhm)
circuit.R(name="bot", positive="vout", negative=circuit.gnd, value=1 @ u_kOhm)

3. Print the Netlist

str(circuit) returns ordinary SPICE text. This is useful for debugging and for workflows where another tool runs the netlist.

print(circuit)
.title voltage_divider
Vin vin 0 10
Rtop vin vout 2k
Rbot vout 0 1k
.end

4. Run a Simulation

Create a simulator from the circuit. Pass a backend name to force one, or omit it to auto-select from installed backends.

sim = circuit.simulator(simulator="ngspice")
op = sim.operating_point()
print(op["vout"])

Common analyses:

GoalCall
DC operating pointsim.operating_point()
DC source sweepsim.dc(Vin=slice(0, 5, 0.1))
AC sweepsim.ac("dec", 100, 1, 1e9)
Transientsim.transient(1e-9, 100e-9)
Noisesim.noise("out", "0", "Vin")

Simulator options are set before the analysis:

sim.options(RELTOL="1e-4")
sim.temperature = 27.0
sim.save("V(vout)", "V(vin)")
sim.save_currents = True

5. Read Results

Operating point results return scalar node values. Sweep and time-domain results return arrays plus an axis.

op = sim.operating_point()
print(op["vout"])

dc = sim.dc(Vin=slice(0, 5, 0.5))
print(dc.sweep)
print(dc["vout"])

ac = sim.ac("dec", 20, 1, 1e6)
print(ac.frequency)
print(ac["vout"])

tran = sim.transient(1e-6, 1e-3)
print(tran.time)
print(tran["vout"])

Most result objects also expose parsed .meas values as result.measures.

6. Use Units

Unit helpers use Python's @ operator and format values as SPICE engineering numbers.

from pyspice_rs.unit import u_V, u_mV, u_kOhm, u_uF

circuit.V(name="bias", positive="vbias", negative=circuit.gnd, value=750 @ u_mV)
circuit.R(name="load", positive="vout", negative=circuit.gnd, value=10 @ u_kOhm)
circuit.C(name="cap", positive="vout", negative=circuit.gnd, value=1 @ u_uF)

7. Add Building Blocks

The core API covers common SPICE elements. Keep node names consistent; shared node strings are electrically connected.

KindMethods
Passive partsR, C, L, K
SourcesV, I, BV, BI
WaveformsSinusoidalVoltageSource, PulseVoltageSource, PieceWiseLinearVoltageSource
Controlled sourcesE, G, F, H
DevicesD, Q/BJT, M/MOSFET, J, Z
Switches and linesS, W, T
Raw SPICEraw_spice, include, lib, parameter, options
 
circuit.PulseVoltageSource(
    name="clk",
    positive="clk",
    negative=circuit.gnd,
    initial_value=0.0,
    pulsed_value=1.8,
    pulse_width=5e-9,
    period=10e-9,
)

circuit.MOSFET(
    name="n1",
    drain="out",
    gate="in",
    source=circuit.gnd,
    bulk=circuit.gnd,
    model="nch",
    W="1u",
    L="130n",
)

8. Add Models and Subcircuits

Semiconductors and switches usually need model definitions. Use inline models, includes, libraries, or raw SPICE when needed.

circuit.model("nch", "NMOS", LEVEL=1, VTO=0.4, KP=200e-6)
circuit.include("/path/to/device_models.spice")
circuit.lib("/path/to/corners.lib", "tt")
circuit.parameter("RLOAD", "1k")

For reusable blocks, use Subcircuit with Testbench. The examples in this repo use that style for larger circuits.

dut = ps.Subcircuit("divider", ["vin", "vout"])
dut.R(name="top", positive="vin", negative="vout", value=2 @ u_kOhm)
dut.R(name="bot", positive="vout", negative=dut.gnd, value=1 @ u_kOhm)

tb = ps.Testbench(dut)
tb.V(name="supply", positive="vin", negative=dut.gnd, value=10 @ u_V)
print(tb.netlist("ngspice"))

9. Choose a Backend

Supported backend names include ngspice, xyce, ltspice, spectre, and vacask. Availability depends on what is installed on your machine.

print(ps.CircuitSimulator.available_backends())

sim = circuit.simulator(simulator="ngspice")
print(sim.check_backend("ngspice"))

Start with ngspice. Use another backend when your circuit or workflow needs it: Xyce for large sweeps and sampling, LTspice for LTspice-oriented flows, Spectre for commercial analog/RF flows, or VacaSk for compact-model workflows.

10. Lint Before Running

The linter catches common netlist problems and some backend-specific issues before you run a simulator.

report = ps.lint(str(circuit), backend="ngspice")

for error in report["errors"]:
    print(error["message"])

for warning in report["warnings"]:
    print(warning["message"])

11. Reuse Testbenches

The optional testbenches package contains small recipes for common design checks. A recipe returns a DesignBench with a normal pyspice_rs.Testbench inside it.

from testbenches import amplifier_voltage_gain

dut = ps.Subcircuit("amp", ["vin", "vout"])
dut.R(name="load", positive="vout", negative=dut.gnd, value=1e6)

bench = amplifier_voltage_gain(ps, dut, input_node="vin", output_node="vout")
print(bench.netlist("ngspice"))
